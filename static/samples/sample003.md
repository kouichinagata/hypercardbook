Page 1:
# HyperCardの歴史
〜世界初のノーコード開発環境が生んだ夢〜

Page 2:
![Macintosh](https://images.unsplash.com/photo-1517694712202-14dd9538aa97?w=600&auto=format&fit=crop)

Page 3:
## はじめに
今では「ノーコード」や「ローコード」という言葉が当たり前になりました。

Page 4:
プログラミングを知らなくてもアプリを作れる。
ドラッグ＆ドロップで画面を配置できる。
ボタンを押せば処理が実行される。

Page 5:
そんな世界は最近始まったように見えますが、実は40年近く前に、その理想を実現していたソフトウェアがありました。

Page 6:
その名は **HyperCard（ハイパーカード）**。

Page 7:
Apple Macintosh向けに開発されたHyperCardは、多くの人に「コンピュータで何かを作る楽しさ」を教えた伝説的なソフトウェアです。

Page 8:
## HyperCard誕生
HyperCardは1987年、Appleによって発表されました。

Page 9:
開発したのはMacintoshチームの中心人物の一人である
**Bill Atkinson**
です。

Page 10:
![Bill Atkinson](https://images.unsplash.com/photo-1547082299-de196ea013d6?w=600&auto=format&fit=crop)

Page 11:
彼はMacPaintやQuickDrawなど、Macintoshの基礎技術を生み出した天才エンジニアとして知られています。

Page 12:
HyperCardは当時のMacintoshに無料で同梱され、多くのユーザーが最初から利用できました。

Page 13:
これは当時としては非常に画期的なことでした。

Page 14:
## カードという発想
HyperCardでは画面を「カード」と呼びました。

Page 15:
カードを何枚も集めたものを「スタック」と呼びます。

Page 16:
![Cards Stack](https://images.unsplash.com/photo-1606166325683-e6deb697d30e?w=600&auto=format&fit=crop)

Page 17:
たとえば、
* 名刺管理
* 図鑑
* 学習教材
* ゲーム
* プレゼンテーション
などを簡単に作ることができました。

Page 18:
現在のWebサイトでいうなら、
* カード = Webページ
* スタック = Webサイト
のような関係です。

Page 19:
## 世界初のハイパーリンク
HyperCardの「Hyper」は、ハイパーテキストの意味です。

Page 20:
ボタンを押すと別のカードへ移動できます。

Page 21:
これは現在のWebブラウザのリンクとほぼ同じ考え方です。

Page 22:
実はHyperCardが登場したのは、
World Wide Web Consortium
によるWebの普及より前でした。

Page 23:
そのため、
**「HyperCardはWebの先祖の一つ」**
とも言われています。

Page 24:
## HyperTalkという魔法
HyperCardには
**HyperTalk**
という独自のスクリプト言語が搭載されていました。

Page 25:
例えばボタンを押した時に、
`[本を生成・更新しました]`
と書くだけでダイアログが表示されます。

Page 26:
プログラミング経験のない人でも理解できるように設計されていました。

Page 27:
英語の文章を読む感覚でプログラムを書けたため、多くの教育現場で利用されました。

Page 28:
## 教育界での大成功
HyperCardは学校で特に人気がありました。

Page 29:
生徒たちは
* クイズ
* 電子図鑑
* 学習教材
* 歴史年表
* 地理情報
などを自分で作成しました。

Page 30:
![Learning](https://images.unsplash.com/photo-1503676260728-1c00da094a0b?w=600&auto=format&fit=crop)

Page 31:
「使う側」だけでなく、
**「作る側になる」**
という体験を、多くの子供たちに与えたのです。

Page 32:
現在のSTEM教育やプログラミング教育の先駆けとも言える存在でした。

Page 33:
## 日本での人気
日本でもHyperCardは大ヒットしました。

Page 34:
当時のMacユーザーの多くが利用しており、
* 教材
* CD-ROMコンテンツ
* ゲーム
* データベース
などが大量に作られました。

Page 35:
特に有名だったのは、
画像の上に透明ボタンを配置し、
クリックした部分の説明を表示する図鑑です。

Page 36:
人体図鑑や機械図鑑などが数多く作られました。

Page 37:
この仕組みは現在でも十分通用するアイデアです。

Page 38:
## HyperCardの衰退
しかし1990年代後半になると状況が変わります。
インターネットの普及です。

Page 39:
人々はHyperCardのスタックではなく、
Webページを作るようになりました。

Page 40:
さらにAppleの経営悪化も重なり、
HyperCardへの投資は徐々に縮小されました。

Page 41:
1998年に開発は事実上終了。

Page 42:
2004年には正式に販売終了となりました。

Page 43:
## HyperCardが残したもの
HyperCardは消えてしまいました。
しかし、その思想は現在も生きています。

Page 44:
例えば、
* Webブラウザ
* Wiki
* Notion
* Scratch
* Airtable
* Bubble
* Glide
* ノーコードツール全般
などにはHyperCardの影響を見ることができます。

Page 45:
「プログラマでなくても作れる」
という思想は、まさにHyperCardが切り開いた世界でした。

Page 46:
## AI時代に復活するHyperCardの思想
近年、生成AIの登場によって状況は再び変わり始めています。

Page 47:
AIに
「クイズアプリを作って」
と頼めば、
HTMLやJavaScriptを自動生成できる時代になりました。

Page 48:
つまり、
HyperCardが目指した
**「誰でもアプリを作れる世界」**
が、さらに進化した形で実現しつつあるのです。

Page 49:
Markdown、AI、ノーコード。

Page 50:
これらは別々の技術に見えますが、
根底にはHyperCardと同じ思想があります。

Page 51:
## おわりに
HyperCardは単なるソフトウェアではありませんでした。

Page 52:
それは、
**「コンピュータは創造の道具である」**
という思想そのものだったのです。

Page 53:
もしHyperCardが存在しなければ、
現在のWebも、 ノーコードも、 生成AIも、
今とは少し違う形になっていたかもしれません。

Page 54:
そして今、AI時代の新しいHyperCardが生まれようとしています。

Page 55:
その候補の一つが、MarkdownとAIを組み合わせた新しい開発環境なのかもしれません。
