Page 1:
## Markdownの歴史

シンプルで強力なマークアップ言語。
その誕生から現在までを辿ります。

Page 2:
### はじめに

Markdownは、ウェブ上でテキストを書く方法を大きく変えました。
その始まりを見ていきましょう。

image: https://picsum.photos/600/400?random=1

Page 3:
### 誕生の背景

2000年代初頭、ウェブでの記述は複雑なHTMLが主流でした。
もっと手軽な方法が求められました。

Page 4:
### 創始者：ジョン・グルーバー

Markdownは、ジョン・グルーバーによって考案されました。
読みやすく、書きやすいことを目指しました。

image: https://picsum.photos/600/400?random=2

Page 5:
### 共同開発：アーロン・スワーツ

彼は初期のMarkdownの構文設計に貢献しました。
シンプルさに磨きをかけました。

Page 6:
### 最初のリリース

2004年、Markdownの最初のバージョンが公開されました。
これは大きな一歩でした。

image: https://picsum.photos/600/400?random=3

Page 7:
### その目的

Markdownの目標は、人間が読みやすいプレーンテキスト形式で記述し、それをHTMLに変換することでした。

Page 8:
### 基本的な考え方

「読みやすさ」が最優先されました。
HTMLタグを直接書くよりもずっと直感的です。

Page 9:
### 初期ユーザー

特にブログ作成者や開発者の間で、その手軽さが評価されました。
すぐに人気を集めました。

image: https://picsum.photos/600/400?random=4

Page 10:
### フレーバーの登場

Markdownの人気に伴い、様々な拡張版（フレーバー）が生まれました。
機能が追加されていきました。

Page 11:
### GFM (GitHub Flavored Markdown)

最も有名なフレーバーの一つです。
GitHubで広く使われ、多くの新機能が加わりました。

video: https://www.youtube.com/watch?v=cM50VjI058I

Page 12:
### CommonMarkの誕生

フレーバーの乱立により、互換性の問題が生じました。
そこで、標準化の動きが始まりました。

Page 13:
### 標準化への道

CommonMarkは、Markdownの明確な仕様を定義するプロジェクトです。
異なるツール間での互換性を高めます。

image: https://picsum.photos/600/400?random=5

Page 14:
### 広がる利用

現在では、ドキュメント作成、READMEファイル、チャットツールなど、多岐にわたる場所で使われています。

Page 15:
### エディタの進化

Markdown専用のエディタも多数登場しました。
より快適な記述環境を提供しています。

Page 16:
### ウェブへの影響

Markdownは、ウェブコンテンツの作成を民主化しました。
誰もが簡単に情報を発信できるようにしました。

image: https://picsum.photos/600/400?random=6

Page 17:
### 未来へ

Markdownは進化を続けています。
そのシンプルさと汎用性は、これからも長く愛されるでしょう。

Page 18:
### まとめ

Markdownの歴史は、シンプルさが勝利した物語です。
今も私たちのデジタルライフを支えています。
