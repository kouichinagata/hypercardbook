# Markdownの歴史

シンプルさが生んだ革命

Page 2:
## Markdownとは？

Markdownは、プレーンテキスト形式で記述できる軽量マークアップ言語です。

Page 3:
とても読みやすく、書きやすいのが特徴です。

Page 4:
## 誕生の背景

インターネットが広がり、HTMLが主流でした。しかし、HTMLは複雑でした。

Page 5:
より簡単にウェブコンテンツを記述する方法が求められていました。

Page 6:
## 開発者たち

Markdownは、ジョン・グルーバーによって考案されました。

Page 7:

ジョン・グルーバー氏

Page 8:
また、アーロン・シュワルツも初期の構文開発に協力しました。

Page 9:
image: https://upload.wikimedia.org/wikipedia/commons/e/e0/Aaron_Swartz_at_MIT_2009.jpg
アーロン・シュワルツ氏

Page 10:
## 誕生の年

Markdownが初めてリリースされたのは、2004年のことです。

Page 11:
今から約20年前のことになります。

Page 12:
## 目的

その主な目的は、「人が読み書きしやすいプレーンテキスト形式」を提供することでした。

Page 13:
そして、それをHTMLに変換できるようにすることです。

Page 14:
## インスピレーション

電子メールの慣習や、setext、Textileといった既存のマークアップ言語から影響を受けました。

Page 15:
特に、記号を使ったシンプルな表現が特徴です。

Page 16:
## 初期普及

ジョン・グルーバー自身のブログ「Daring Fireball」で利用され、注目を集めました。

Page 17:
開発者コミュニティを中心に、徐々に利用者が増えていきました。

Page 18:
## 広がる活用

GitHubが「GitHub Flavored Markdown (GFM)」を導入し、さらに普及が加速。

Page 19:
ドキュメント作成やREADMEファイルに欠かせないものとなりました。

Page 20:
## 標準化の動き

様々な「フレーバー」が乱立したため、CommonMarkという標準化の動きも生まれました。

Page 21:
これにより、Markdownの互換性が向上しました。

Page 22:
## 今日のMarkdown

現在、Markdownは開発者だけでなく、多くの人に利用されています。

Page 23:
メモ、ブログ、書籍執筆、プレゼンテーションなど、用途は多岐にわたります。

Page 24:
# 終わり

シンプルさが生んだ、偉大なテキストフォーマットの物語。
